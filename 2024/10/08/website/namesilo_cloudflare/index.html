Namesilo注册域名并使用Cloudflare进行解析

Created2024-10-08|Updated2024-10-10|网站建设服务器配置
|Post Views:

1. 在 Namesilo 注册域名

为什么选择 namesilon ，而不选择国内阿里云之类的注册呢，因为便宜啊，后面续费都是按照第一年的价格，不像国内的厂商第一年很便宜，后面续费逐级递增

访问 Namesilo官网。

在首页的搜索框中输入您想要的域名，并点击“SEARCH DOMIAN”。可以看到有许多后缀供你选择，直接选最便宜的就好了

选好后，点击“Checkout”。

完成付款流程，支持支付宝付费。

域名注册成功后，您可以在 Namesilo 的管理控制面板中看到你·的域名。

2. 将域名指向 Cloudflare

域名和互联网上的服务器是通过IP地址进行连接的。域名解析是将易于记忆的域名（如 www.xxxx.com）转换成服务器IP地址（如 192.0.2.1）的过程。

Namesilo 进入域名管理页面。

找到你刚注册的域名，点击上方的”Change Nameservers”。

进入后可以看到 namesilo 自动给你分配了三个名称服务器，因为我们要托管到 Cloudflare，所以我们要删掉默认名称服务器，更改为 Cloudflare 的名称服务器

下面我们登录 [Cloudflare官网](科赋锐信息科技Cloudflare | Cloudflare (cloudflare-cn.com))，输入之前的域名，点击”继续”

然后进入薅羊毛环节，这肯定选择免费，free版对我们也够用了。

Cloudflare 提供以下两个名称服务器：

将这两个名称服务器复制到 Namesilo 的DNS管理页面，并保存更改。

3. 在 Cloudflare 配置域名解析

Namesilo有DNS解析，但是为什么要用Cloudflare，据说是功能更强大一点，这我也不知道，如果嫌在cloudflare麻烦就看这篇博客：3步搞定Namesilo域名解析_namesilo域名解析教程-CSDN博客

回到 Cloudflare 的管理面板，可以看到 Namesilo 自动帮我们添加两条记录，当然我们需要自己配置，这两条删去就行

在“DNS”标签页下，您可以添加、编辑或删除DNS记录。例如，添加A记录指向您的服务器IP地址，或者添加CNAME记录指向其他服务。

主机记录 解释
www 解析后的域名为 www.zbs666.top
@ 直接解析主域名 zbs666.top
* 泛解析，匹配其他所有域名 *.zbs666.top
mail 将域名解析为 mail.zbs666.top，通常用于解析邮箱服务器
二级 如 abc.zbs666.top，填写 abc
手机网站 如 m.zbs666.top，填写 m
显性URL 不支持泛解析（泛解析：将所有子域名解析到同一地址）
记录类型 解释
A 用来指定域名的 IPv4 地址（如 8.8.8.8），如果需要将域名指向一个 IP 地址，就需要添加 A 记录。
CNAME 如果需要将域名指向另一个域名，再由另一个域名提供 IP 地址，就需要添加 CNAME 记录。
MX 如果需要设置邮箱，让邮箱能收到邮件，就需要添加 MX 记录。
TXT 在这里可以填写任何东西，长度限制 255。绝大多数的 TXT 记录是用来做 SPF 记录（反垃圾邮件）
NS 域名服务器记录，如果需要将子域名交给其他 DNS 服务商解析，就需要添加 NS 记录。
AAAA 用来指定主机名（或域名）对应的 IPv6 地址（例如：ff06:0:0:0:0:0:0:c3）记录。
SRV 记录了哪台计算机提供了哪个服务。格式为：服务的名字、点、协议的类型，例如：_xmpp-server_tcp。
显性URL 从一个地址 301 重定向到另一个地址的时候，就需要添加显性 URL 记录（注：DNSPod 目前只支持 301 重定向）。
隐性URL 类似于显性 URL，区别在于隐性 URL 不会改变地址栏的域名。

按照需求进行配置就好了，等待一段时间（通常几分钟到几小时)，让DNS记录在全球范围内传播。
